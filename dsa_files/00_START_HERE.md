## 🌳 The Flowchart Feature

### What It Does
Helps you identify which pattern to use by asking simple questions.

### How It Works
```
Question 1: "What type of data structure?"
        ↓
Question 2: "What's the task?"
        ↓
Question 3: "Any specific constraints?"
        ↓
Result: "✓ Use Pattern XYZ"
Explanation + link to details
```

| # | Pattern | Type | Best For |
|---|---------|------|----------|
| 1 | Sliding Window | Array/String | Contiguous sequences, max/min |
| 2 | Two Pointers | Array | Pairs, triplets, palindromes |
| 3 | Fast & Slow Pointers | LinkedList | Cycle detection |
| 4 | Binary Search | Sorted Array | Searching, O(log n) |
| 5 | BFS | Graph/Tree | Shortest path, distance |
| 6 | DFS | Graph/Tree | All paths, components |
| 7 | Dynamic Programming | Any | Optimization, overlapping subproblems |
| 8 | Backtracking | Any | All solutions, permutations |
| 9 | Heap / Priority Queue | Any | Top K elements |
| 10 | Union-Find | Graph | Connected components |
| 11 | Monotonic Stack | Array | Next greater/smaller |
| 12 | Prefix Sum | Array | Range queries |
| 13 | Trie | String | Prefix search |
| 14 | Topological Sort | Graph | Dependencies |
