Phase 1 Core Linear Patterns — Weeks 1–2
Build Time
O(n)
Query Time
O(1)
Space
O(n)
When to Use

Strong Signals ✓

  • Multiple range sum queries on the same static array
  • Subarray sum equals K — extremely common interview problem
  • 2D grid rectangle sum queries
  • Subarrays with sum divisible by K (modulo prefix)
  • Any problem where you need accumulated information from index 0

Avoid When ✗

  • Array is modified after build → Segment Tree or BIT
  • Only a single query needed → just iterate once
  • Need maximum subarray sum → Kadane's Algorithm
  • Sliding window of fixed length → Sliding Window pattern
Variants
Build prefix[i] = sum of nums[0..i-1]. Range sum [l,r] = prefix[r+1] - prefix[l] in O(1). Always initialize prefix[0] = 0 as the empty-prefix base case.
C#
public class NumArray
{
    private readonly int[] _prefix;

    public NumArray(int[] nums)
    {
        _prefix = new int[nums.Length + 1];
        for (int i = 0; i < nums.Length; i++)
            _prefix[i + 1] = _prefix[i] + nums[i];
    }

    // O(1): sum from index left to right inclusive
    public int SumRange(int left, int right)
        => _prefix[right + 1] - _prefix[left];
}
Don't store the prefix array. Use a HashMap of seen prefix sums. When you reach index i with currentSum, check if (currentSum - k) was seen. If so, there's a subarray ending at i with sum k. Handles negatives perfectly.
C#
public int SubarraySum(int[] nums, int k)
{
    var prefixCount = new Dictionary<int, int>();
    prefixCount[0] = 1; // empty prefix (sum=0 seen once)
    int curr = 0, count = 0;

    foreach (int num in nums)
    {
        curr += num;
        // subarrays ending here with sum k
        count += prefixCount.GetValueOrDefault(curr - k, 0);
        prefixCount[curr] = prefixCount.GetValueOrDefault(curr, 0) + 1;
    }
    return count;
}
Extend 1D to 2D. Query any rectangle with inclusion-exclusion formula. Build is O(m*n), each query is O(1). Memorize the formula: p[r2+1][c2+1] - p[r1][c2+1] - p[r2+1][c1] + p[r1][c1].
C#
public class NumMatrix
{
    private readonly int[,] _p;
    public NumMatrix(int[][] m)
    {
        int R = m.Length, C = m[0].Length;
        _p = new int[R+1, C+1];
        for (int r=1; r<=R; r++)
        for (int c=1; c<=C; c++)
            _p[r,c] = m[r-1][c-1]+_p[r-1,c]+_p[r,c-1]-_p[r-1,c-1];
    }
    public int SumRegion(int r1,int c1,int r2,int c2)
        => _p[r2+1,c2+1]-_p[r1,c2+1]-_p[r2+1,c1]+_p[r1,c1];
}
Decision Guide

Use this pattern when

  • Multiple range queries on static array
  • Count subarrays where sum = k exactly
  • Subarrays divisible by k (prefix % k trick)
  • 2D rectangle sum queries

Prefer another when

  • Array changes → Binary Indexed Tree or Segment Tree
  • Max subarray sum → Kadane's Algorithm
  • Fixed-length window → Sliding Window
  • Single query → just iterate
If Pattern Does Not Apply — Pivot Thinking

Decision Framework

Q1Does the array change? → BIT for point updates, Segment Tree for range updates — both support O(log n) modifications.
Q2Is the window fixed-length? → Sliding Window handles it in O(1) space without building a prefix array.
Q3Need the longest/shortest subarray satisfying a condition? → Two Pointers or Sliding Window + condition tracking.
Q42D grid with connectivity or path queries? → BFS/DFS, not prefix sum.

Time
O(n)
Space
O(1) or O(k)
Data
Array / String
When to Use

Strong Signals ✓

  • Subarray or substring of fixed size k
  • Longest subarray/substring satisfying a condition
  • Minimum window containing all required elements
  • Contiguous elements only — not subsequences
  • Optimize a property (sum, distinct count) over all windows

Avoid When ✗

  • Non-contiguous subsequences → Two Pointers or DP
  • Need prefix sums across multiple queries → Prefix Sum + HashMap
  • All pairs not just adjacent → Two Pointers
  • Need max/min inside each window → add Monotonic Deque
Variants
Window size k is constant. Compute the first window, then slide: add the right element, subtract the left element. O(1) per step after initialization.
C#
public int MaxSumFixed(int[] nums, int k)
{
    int windowSum = 0, maxSum = int.MinValue;

    for (int i = 0; i < nums.Length; i++)
    {
        windowSum += nums[i];

        if (i >= k - 1)
        {
            maxSum = Math.Max(maxSum, windowSum);
            windowSum -= nums[i - k + 1]; // remove oldest element
        }
    }
    return maxSum;
}
Find the longest window satisfying a condition. Expand right unconditionally. Shrink left only when the condition is violated. Track max length at each valid state.
C#
public int LengthOfLongestSubstring(string s)
{
    var seen = new Dictionary<char, int>(); // char → last index
    int left = 0, maxLen = 0;

    for (int right = 0; right < s.Length; right++)
    {
        char c = s[right];
        // if char seen inside current window, jump left past it
        if (seen.TryGetValue(c, out int last) && last >= left)
            left = last + 1;

        seen[c] = right;
        maxLen = Math.Max(maxLen, right - left + 1);
    }
    return maxLen;
}
Expand right until satisfied, then shrink left as far as possible while still satisfied. Record minimum at each valid state. Classic: Minimum Window Substring.
C#
public string MinWindow(string s, string t)
{
    var need = new Dictionary<char, int>();
    foreach (char c in t) need[c] = need.GetValueOrDefault(c) + 1;

    int left=0, formed=0, required=need.Count, minLen=int.MaxValue, minL=0;
    var window = new Dictionary<char, int>();

    for (int r = 0; r < s.Length; r++)
    {
        char c = s[r];
        window[c] = window.GetValueOrDefault(c) + 1;
        if (need.ContainsKey(c) && window[c] == need[c]) formed++;

        while (formed == required)
        {
            if (r - left + 1 < minLen) { minLen = r - left + 1; minL = left; }
            char lc = s[left++];
            window[lc]--;
            if (need.ContainsKey(lc) && window[lc] < need[lc]) formed--;
        }
    }
    return minLen == int.MaxValue ? "" : s.Substring(minL, minLen);
}
Track the maximum or minimum inside a sliding window using a monotonic deque (LinkedList). Deque stores indices in decreasing value order for max. Each element enters and leaves the deque exactly once: O(n) total.
C#
public int[] MaxSlidingWindow(int[] nums, int k)
{
    var res = new int[nums.Length - k + 1];
    var dq = new LinkedList<int>(); // indices, decreasing values
    int ri = 0;

    for (int i = 0; i < nums.Length; i++)
    {
        // remove indices outside window
        while (dq.Count > 0 && dq.First.Value < i - k + 1)
            dq.RemoveFirst();
        // maintain decreasing order
        while (dq.Count > 0 && nums[dq.Last.Value] < nums[i])
            dq.RemoveLast();
        dq.AddLast(i);
        if (i >= k - 1) res[ri++] = nums[dq.First.Value];
    }
    return res;
}
Decision Guide

Use this pattern when

  • Subarray/substring must be contiguous
  • Fixed window: sum/average of every k elements
  • Variable window: longest with at most k distinct chars
  • Minimum window enclosing all required chars

Prefer another when

  • Non-contiguous subsequences → Two Pointers or DP
  • Need prefix sum info across queries → Prefix Sum + HashMap
  • Negative numbers + max subarray → Kadane's (negatives break shrink monotonicity)
  • All subarrays of exactly k length with count → at-most-k trick + Prefix Sum
If Pattern Does Not Apply — Pivot Thinking

Decision Framework

Q1Non-contiguous elements? → Two Pointers (sorted) or DP (subsequences). Key difference: sliding window is one contiguous segment; Two Pointers has two independent cursors.
Q2Need count of subarrays? → Use the 'at most k' trick: answer(k) - answer(k-1) = exactly k subarrays. Combine with prefix sum HashMap.
Q3Negative numbers and need max sum? → Kadane's Algorithm. Negatives break the expand/shrink monotonicity that sliding window relies on.

Sorted or partitioned
When to Use

Strong Signals ✓

  • Sorted array — find pairs/triplets summing to target
  • Palindrome checks or reversal
  • In-place partition (move zeros, Dutch Flag)
  • Remove duplicates from sorted array
  • Merging two sorted arrays

Avoid When ✗

  • Unsorted array where sorting loses index info → HashMap
  • Contiguous subarray needed → Sliding Window
  • Working with a linked list → Fast & Slow Pointers
  • Need count of all subarrays → Prefix Sum + HashMap
Variants
Pointers start at opposite ends, move toward center. The sorted property tells you which direction to move: if sum is too small, advance left; if too large, advance right. Works for two-sum on sorted arrays, container with most water, palindrome check.
C#
public int[] TwoSum(int[] nums, int target)
{
    int left = 0, right = nums.Length - 1;
    while (left < right)
    {
        int sum = nums[left] + nums[right];
        if      (sum == target) return new[] { left+1, right+1 };
        else if (sum < target)  left++;
        else                    right--;
    }
    return new[] { -1, -1 };
}
Fix one element with a loop, apply two-pointer on the remaining sorted subarray. Skip duplicates at both the fixed index and after each match. Pattern generalizes: 4Sum fixes two outer elements.
C#
public IList<IList<int>> ThreeSum(int[] nums)
{
    Array.Sort(nums);
    var res = new List<IList<int>>();

    for (int i = 0; i < nums.Length - 2; i++)
    {
        if (i > 0 && nums[i] == nums[i-1]) continue; // skip dup fixed
        int l = i+1, r = nums.Length-1;
        while (l < r)
        {
            int sum = nums[i]+nums[l]+nums[r];
            if (sum == 0)
            {
                res.Add(new[] {nums[i],nums[l],nums[r]});
                while (l<r && nums[l]==nums[l+1]) l++;
                while (l<r && nums[r]==nums[r-1]) r--;
                l++; r--;
            }
            else if (sum < 0) l++;
            else            r--;
        }
    }
    return res;
}
Both pointers start at the same end. Slow marks the write position, fast scans ahead. Used for in-place manipulation: remove duplicates, move zeros, Dutch National Flag (3-way partition).
C#
// Move Zeros — slow = write pos, fast = scanner
public void MoveZeroes(int[] nums)
{
    int slow = 0;
    for (int fast = 0; fast < nums.Length; fast++)
        if (nums[fast] != 0) nums[slow++] = nums[fast];
    while (slow < nums.Length) nums[slow++] = 0;
}

// Dutch National Flag (sort 0s, 1s, 2s) — 3 pointers
public void SortColors(int[] nums)
{
    int lo = 0, mid = 0, hi = nums.Length - 1;
    while (mid <= hi)
    {
        if      (nums[mid] == 0) (nums[lo++], nums[mid++]) = (nums[mid], nums[lo]);
        else if (nums[mid] == 1) mid++;
        else                     (nums[mid], nums[hi--]) = (nums[hi], nums[mid]);
    }
}
Decision Guide

Use this pattern when

  • Input is sorted or can be sorted without losing needed info
  • Looking for pairs/triplets summing to target
  • In-place partition or duplicate removal needed
  • Palindrome verification or two-end squeeze

Prefer another when

  • Need original indices → use HashMap instead of sorting
  • Contiguous subarray → Sliding Window
  • Linked list → Fast & Slow Pointers (no random access)
  • All pairs including non-adjacent → nested loop or Prefix Sum
If Pattern Does Not Apply — Pivot Thinking

Decision Framework

Q1Array not sorted and sorting loses info? → HashMap complement lookup. Two Sum I solves in O(n) with no sorting.
Q2About contiguous windows, not pairs? → Sliding Window. Key difference: window is one segment; two pointers are two independent cursors squeezing.
Q3Linked list? → Fast & Slow Pointers adapts two-pointer for structures with no random access.
Q43Sum but duplicates in output are OK? → No need to skip duplicate fixed element. Simplifies significantly.

Structure
Linked List / Sequence
When to Use

Strong Signals ✓

  • Detect cycle in linked list
  • Find start/entry point of a cycle
  • Find middle of linked list in one pass
  • Kth node from end without counting length first
  • Happy Number — detect cycle in integer sequence

Avoid When ✗

  • Array with random access → two-pointer is simpler
  • Graph cycle detection → DFS with 3-color marking
  • Tree middle → BFS level-order is cleaner
  • Need cycle length explicitly → count steps between meetings
Variants
Slow moves 1 step, fast moves 2. Meeting point proves a cycle. To find cycle entry: reset slow to head, advance both 1 step at a time — they meet exactly at the cycle's entry node. Mathematical proof uses modular arithmetic on cycle lengths.
C#
// Phase 1: detect. Phase 2: find entry.
public ListNode DetectCycle(ListNode head)
{
    ListNode slow = head, fast = head;

    while (fast != null && fast.next != null)
    {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) break;
    }

    if (fast == null || fast.next == null) return null;

    // Reset slow to head; advance both 1 step
    slow = head;
    while (slow != fast) { slow = slow.next; fast = fast.next; }
    return slow; // cycle entry node
}
When fast reaches the end, slow is at the middle. For kth-from-end: advance fast k steps first, then move both together — when fast hits null, slow is at the kth node from end. O(n) single pass.
C#
public ListNode MiddleNode(ListNode head)
{
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null)
    {
        slow = slow.next;
        fast = fast.next.next;
    }
    return slow; // second middle for even-length lists
}

// Kth from end: move fast k steps ahead first
public ListNode KthFromEnd(ListNode head, int k)
{
    ListNode slow = head, fast = head;
    for (int i = 0; i < k; i++) fast = fast.next;
    while (fast != null) { slow = slow.next; fast = fast.next; }
    return slow;
}
Apply Floyd's cycle detection where the 'next pointer' is replaced by the digit-square-sum function f(n). If the sequence reaches 1, it's happy. If slow meets fast before reaching 1, there's a loop and it's not happy. No HashSet needed.
C#
public bool IsHappy(int n)
{
    int slow = n, fast = Next(n);
    while (fast != 1 && slow != fast)
    {
        slow = Next(slow);
        fast = Next(Next(fast));
    }
    return fast == 1;
}

int Next(int n)
{
    int sum = 0;
    while (n > 0) { int d = n % 10; sum += d*d; n /= 10; }
    return sum;
}
Decision Guide

Use this pattern when

  • Linked list cycle detection in O(1) space
  • Find middle without computing length first
  • Kth from end in one pass
  • Sequence cycle in integer problems

Prefer another when

  • Graph cycle → DFS with 3 colors (white/gray/black)
  • Directed graph → Topological Sort (Kahn's detects cycles)
  • Array cycle detection → visited set or index negation trick
  • Need cycle length → count steps between slow and fast meeting
If Pattern Does Not Apply — Pivot Thinking

Decision Framework

Q1Graph (not linked list) cycles? → DFS with visited states. Fast & Slow only works when there's a well-defined 'next pointer', not an adjacency list.
Q2Cycle in a directed graph for dependency resolution? → Topological Sort (Kahn's algorithm) — it naturally detects cycles when the result count < n.

Phase 2 Search & Divide — Weeks 2–3
Time
O(log n)
Space
O(1)
Requires
Sorted or monotonic property
When to Use

Strong Signals ✓

  • Sorted array — find an element or its position
  • Find first/last position satisfying a condition
  • Rotated sorted array search
  • Minimize/maximize with a feasibility function (binary search on answer)
  • Any problem where brute force is O(n) and the answer space is monotonic

Avoid When ✗

  • Unsorted with no monotonic property → HashMap or linear scan
  • Need all occurrences (use upper/lower bound difference)
  • 2D matrix with complex layout → BFS or row/column binary search
  • Feasibility function is not monotonic → DP
Variants
The universal template: left=0, right=n-1, while left≤right. Compute mid safely with left+(right-left)/2. This avoids integer overflow. For first/last position use the lower-bound variant with half-open interval.
C#
// Exact match — classic template
public int BinarySearch(int[] nums, int target)
{
    int left = 0, right = nums.Length - 1;
    while (left <= right)
    {
        int mid = left + (right - left) / 2; // overflow-safe
        if      (nums[mid] == target) return mid;
        else if (nums[mid] < target)  left  = mid + 1;
        else                          right = mid - 1;
    }
    return -1;
}

// First position ≥ target (lower bound)
public int LowerBound(int[] nums, int target)
{
    int left = 0, right = nums.Length; // half-open [0, n)
    while (left < right)
    {
        int mid = left + (right - left) / 2;
        if (nums[mid] < target) left = mid + 1;
        else                    right = mid;
    }
    return left; // first index where nums[i] >= target
}
Even after rotation, one half is always sorted. Check which half is sorted by comparing mid to left. Then check if target falls in the sorted half — if yes, search there; otherwise search the other half.
C#
public int SearchRotated(int[] nums, int target)
{
    int left = 0, right = nums.Length - 1;
    while (left <= right)
    {
        int mid = left + (right - left) / 2;
        if (nums[mid] == target) return mid;

        if (nums[left] <= nums[mid]) // left half is sorted
        {
            if (nums[left] <= target && target < nums[mid]) right = mid - 1;
            else                                               left  = mid + 1;
        }
        else // right half is sorted
        {
            if (nums[mid] < target && target <= nums[right]) left  = mid + 1;
            else                                              right = mid - 1;
        }
    }
    return -1;
}
The array itself isn't sorted, but the answer space is monotonic. Binary search on the answer value. Define isOk(mid) as a feasibility function. Find minimum mid where isOk is true (or maximum where it's false). Template: left=minPossible, right=maxPossible, while left<right.
C#
// Koko Eating Bananas — minimize eating speed
public int MinEatingSpeed(int[] piles, int h)
{
    int left = 1, right = piles.Max();

    while (left < right)
    {
        int mid = left + (right - left) / 2;
        if (CanFinish(piles, mid, h)) right = mid;   // try smaller
        else                           left  = mid + 1;
    }
    return left;
}

bool CanFinish(int[] piles, int speed, int h)
{
    int hours = 0;
    foreach (int p in piles)
        hours += (p + speed - 1) / speed; // ceiling division
    return hours <= h;
}

// Pattern: isOk is monotonic — once true, stays true.
// Find minimum where isOk=true: use right=mid when ok, left=mid+1 when not.
Decision Guide

Use this pattern when

  • Sorted input — O(log n) lookup
  • First/last position of an element
  • Minimize/maximize with monotonic feasibility function
  • Any problem where halving the search space is valid

Prefer another when

  • Unsorted, no monotonic property → HashMap or linear scan
  • 2D matrix with row/col independence → binary search each row
  • No feasibility function boundary → DP or brute force
  • Feasibility function is O(n²) → total O(n² log n) might be too slow
If Pattern Does Not Apply — Pivot Thinking

Decision Framework

Q1Array not sorted? Ask: is there a monotonic condition in the problem statement? 'Minimum X that satisfies Y' almost always means binary search on the answer even when the input array isn't sorted.
Q2Feasibility function too slow? Binary search on answer is only worthwhile if the feasibility check is O(n) or O(n log n). If it's O(n²), the total cost may exceed brute force.
Q3Need exact element positions? → Use lower_bound and upper_bound variants. Count occurrences as upper_bound - lower_bound.

Phase 3 Graph Traversal — Weeks 3–4
Time
O(V + E)
Space
O(V) queue
Guarantee
Shortest (unweighted)
When to Use

Strong Signals ✓

  • Shortest path in unweighted graph or grid
  • Minimum steps/moves to reach target
  • Level-order traversal of a tree
  • Nearest X problems (rotten oranges, walls and gates)
  • Multi-source spreading — distances from all sources simultaneously

Avoid When ✗

  • Weighted edges → Dijkstra's (priority queue BFS)
  • Negative weights → Bellman-Ford
  • Just checking connectivity → DFS is simpler
  • All paths or combinations needed → DFS + backtracking
Variants
Queue-based. Enqueue start, mark visited. Loop: dequeue, process neighbors, enqueue unvisited. Track distance by level — capture queue.Count at the start of each level iteration.
C#
public int ShortestPath(int[][] grid, (int r,int c) src, (int r,int c) dst)
{
    int m = grid.Length, n = grid[0].Length;
    var q = new Queue<(int,int)>();
    var vis = new bool[m, n];
    int[] dr = {-1,1,0,0}, dc = {0,0,-1,1};

    q.Enqueue(src); vis[src.r,src.c] = true;
    int steps = 0;

    while (q.Count > 0)
    {
        int size = q.Count; // process one full level
        for (int i = 0; i < size; i++)
        {
            var (r,c) = q.Dequeue();
            if ((r,c)==dst) return steps;
            for (int d=0; d<4; d++)
            {
                int nr=r+dr[d], nc=c+dc[d];
                if (nr>=0&&nr<m&&nc>=0&&nc<n&&!vis[nr,nc]&&grid[nr][nc]!=1)
                { vis[nr,nc]=true; q.Enqueue((nr,nc)); }
            }
        }
        steps++;
    }
    return -1;
}
Start BFS from ALL source nodes simultaneously. Enqueue every source before the first iteration. The queue naturally spreads outward — each cell gets the distance from its nearest source. Classic: Rotten Oranges, 01 Matrix, Walls and Gates.
C#
public int OrangesRotting(int[][] grid)
{
    var q = new Queue<(int,int)>();
    int fresh=0, m=grid.Length, n=grid[0].Length;

    for (int r=0;r<m;r++) for (int c=0;c<n;c++)
    {
        if (grid[r][c]==2) q.Enqueue((r,c)); // all rotten = sources
        if (grid[r][c]==1) fresh++;
    }

    int[] dr={-1,1,0,0}, dc={0,0,-1,1};
    int minutes = 0;

    while (q.Count>0 && fresh>0)
    {
        minutes++;
        int sz=q.Count;
        for (int i=0;i<sz;i++)
        {
            var (r,c)=q.Dequeue();
            for (int d=0;d<4;d++)
            {
                int nr=r+dr[d], nc=c+dc[d];
                if (nr>=0&&nr<m&&nc>=0&&nc<n&&grid[nr][nc]==1)
                { grid[nr][nc]=2; fresh--; q.Enqueue((nr,nc)); }
            }
        }
    }
    return fresh==0 ? minutes : -1;
}
The 'graph' is implicit. Nodes are states (strings, tuples, board configs), edges are valid transitions. BFS finds the shortest transformation sequence. Neighbors are computed on the fly, not stored. Classic: Word Ladder.
C#
public int LadderLength(string begin, string end, IList<string> words)
{
    var wordSet = new HashSet<string>(words);
    if (!wordSet.Contains(end)) return 0;

    var q = new Queue<string>();
    q.Enqueue(begin); wordSet.Remove(begin);
    int steps = 1;

    while (q.Count > 0)
    {
        int sz = q.Count;
        for (int i=0;i<sz;i++)
        {
            char[] w = q.Dequeue().ToCharArray();
            for (int j=0;j<w.Length;j++)
            {
                char orig=w[j];
                for (char ch='a';ch<='z';ch++)
                {
                    w[j]=ch;
                    string next=new string(w);
                    if (next==end) return steps+1;
                    if (wordSet.Remove(next)) q.Enqueue(next);
                }
                w[j]=orig;
            }
        }
        steps++;
    }
    return 0;
}
Decision Guide

Use this pattern when

  • Shortest path guaranteed (BFS processes nodes in order of distance)
  • Level-order tree processing
  • Multi-source spreading problems
  • State space search — implicit graph

Prefer another when

  • Weighted edges → Dijkstra (priority queue BFS)
  • Connectivity or component counting only → DFS is simpler and cleaner
  • All paths or backtracking required → DFS

void Dfs(char[][] g, int r, int c)
{
    if (r<0||r>=g.Length||c<0||c>=g[0].Length||g[r][c]!='1') return;
    g[r][c] = '0'; // mark visited by sinking
    Dfs(g,r+1,c); Dfs(g,r-1,c); Dfs(g,r,c+1); Dfs(g,r,c-1);

Build a current path list as you recurse. When you hit the base case (destination), add a copy of the current path to results. Remove the last element when returning (backtrack). This is where DFS and Backtracking meet.
C#
public IList<IList<int>> AllPathsSourceTarget(int[][] graph)
{
    var result = new List<IList<int>>();
    var path   = new List<int> { 0 };
    Dfs(0);
    return result;

    void Dfs(int node)
    {
        if (node == graph.Length-1)
        { result.Add(new List<int>(path)); return; }
        foreach (int next in graph[node])
        {
            path.Add(next);
            Dfs(next);
            path.RemoveAt(path.Count-1); // backtrack
        }
    }
}
Color nodes: 0=unvisited (WHITE), 1=in-progress (GRAY), 2=done (BLACK). If you encounter a GRAY node during DFS, there's a back edge = cycle. After processing all neighbors, color BLACK. This correctly handles directed graphs.
C#
public bool CanFinish(int n, int[][] prereqs)
{
    var adj = new List<int>[n];
    for (int i=0;i<n;i++) adj[i]=new List<int>();
    foreach (var p in prereqs) adj[p[1]].Add(p[0]);

    int[] color = new int[n]; // 0=WHITE, 1=GRAY, 2=BLACK

    for (int i=0;i<n;i++)
        if (color[i]==0 && HasCycle(i)) return false;
    return true;

    bool HasCycle(int u)
    {
        color[u]=1;
        foreach (int v in adj[u])
        {
            if (color[v]==1) return true; // back edge!
            if (color[v]==0 && HasCycle(v)) return true;
        }
        color[u]=2;
        return false;
    }
}

Every backtracking problem follows: Choose (add to state), Explore (recurse), Unchoose (remove from state). Subsets: start index at i+1 (no reuse). Combinations with repetition: start at i. Permutations: use a used[] array.
C#
// SUBSETS — record at every node
void Subsets(int[] nums, int start, List<int> cur, List<IList<int>> res)
{
    res.Add(new List<int>(cur));
    for (int i=start; i<nums.Length; i++)
    {
        cur.Add(nums[i]);                         // choose
        Subsets(nums, i+1, cur, res);             // explore
        cur.RemoveAt(cur.Count-1);               // unchoose
    }
}

// PERMUTATIONS — all elements, order matters
void Permute(int[] nums, bool[] used, List<int> cur, List<IList<int>> res)
{
    if (cur.Count==nums.Length) { res.Add(new List<int>(cur)); return; }
    for (int i=0; i<nums.Length; i++)
    {
        if (used[i]) continue;
        used[i]=true;  cur.Add(nums[i]);
        Permute(nums, used, cur, res);
        used[i]=false; cur.RemoveAt(cur.Count-1);
    }
}

// COMBINATIONS SUM — reuse allowed: start at i, not i+1
void CombSum(int[] candidates, int target, int start, List<int> cur, List<IList<int>> res)
{
    if (target==0) { res.Add(new List<int>(cur)); return; }
    for (int i=start; i<candidates.Length; i++)
    {
        if (candidates[i]>target) break; // pruning (requires sorted)
        cur.Add(candidates[i]);
        CombSum(candidates, target-candidates[i], i, cur, res); // i not i+1
        cur.RemoveAt(cur.Count-1);
    }
}

public int TotalNQueens(int n)
{
    var cols  = new HashSet<int>();

    var diag2 = new HashSet<int>(); // row+col
    int count = 0;
    Bt(0);
    return count;

    void Bt(int row)
    {
        if (row==n) { count++; return; }
        for (int col=0; col<n; col++)
        {
            if (cols.Contains(col)||diag1.Contains(row-col)||diag2.Contains(row+col)) continue;
            cols.Add(col); diag1.Add(row-col); diag2.Add(row+col);
            Bt(row+1);

        }
    }
}

public bool Exist(char[][] board, string word)

        if (Bt(r,c,0)) return true;
    return false;

    {

        if (r<0||r>=m||c<0||c>=n||board[r][c]!=word[idx]) return false;

State is a single index. dp[i] depends on dp[i-1] or dp[i-2]. Often reducible to O(1) space by keeping only the last 1-2 values. The three building blocks: Fibonacci recurrence, House Robber (skip or take), Climbing Stairs (combine).
C#
// House Robber: dp[i] = max(skip=dp[i-1], take=dp[i-2]+nums[i])
public int Rob(int[] nums)
{
    int prev2=0, prev1=0;
    foreach (int num in nums)
    {
        int curr = Math.Max(prev1, prev2+num);
        prev2=prev1; prev1=curr;
    }
    return prev1;
}

// LIS: dp[i] = longest increasing subseq ending at i
public int LengthOfLIS(int[] nums)
{
    int n=nums.Length; int[] dp=new int[n]; Array.Fill(dp,1);
    for (int i=1;i<n;i++)
        for (int j=0;j<i;j++)
            if (nums[j]<nums[i]) dp[i]=Math.Max(dp[i], dp[j]+1);
    return dp.Max();
}
State needs two indices. dp[i][j] typically depends on dp[i-1][j], dp[i][j-1], or dp[i-1][j-1]. Used for LCS, Edit Distance, Unique Paths, Coin Change 2D. Base cases fill first row/column.
C#
// Edit Distance: min operations to convert word1 to word2
public int MinDistance(string s, string t)
{
    int m=s.Length, n=t.Length;
    int[,] dp=new int[m+1,n+1];
    for (int i=0;i<=m;i++) dp[i,0]=i;
    for (int j=0;j<=n;j++) dp[0,j]=j;

    for (int i=1;i<=m;i++)
    for (int j=1;j<=n;j++)
        dp[i,j] = s[i-1]==t[j-1]
            ? dp[i-1,j-1]
            : 1+Math.Min(dp[i-1,j-1], Math.Min(dp[i-1,j], dp[i,j-1]));
    return dp[m,n];
}
0/1 Knapsack: each item at most once → iterate capacity backwards (prevent reuse in same pass). Unbounded Knapsack: items reusable → iterate capacity forwards. Coin Change is unbounded; 0/1 Knapsack uses backward iteration.
C#
// Coin Change — Unbounded (each coin reusable)
public int CoinChange(int[] coins, int amount)
{
    int[] dp = new int[amount+1];
    Array.Fill(dp, amount+1); // sentinel infinity
    dp[0] = 0;
    foreach (int coin in coins)
        for (int a=coin; a<=amount; a++) // forward = reuse allowed
            dp[a] = Math.Min(dp[a], dp[a-coin]+1);
    return dp[amount]>amount ? -1 : dp[amount];
}

// 0/1 Knapsack — each item at most once
public int Knapsack01(int[] weights, int[] values, int cap)
{
    int[] dp = new int[cap+1];
    foreach (int i in Enumerable.Range(0, weights.Length))
        for (int w=cap; w>=weights[i]; w--) // backward = no reuse
            dp[w] = Math.Max(dp[w], dp[w-weights[i]]+values[i]);
    return dp[cap];
}
dp[i][j] = answer for subarray [i,j]. Iterate by increasing length. Split point k divides interval into two already-solved subproblems. Used in Burst Balloons, Matrix Chain, Palindrome Partitioning II.
C#
// Burst Balloons — dp[i][j] = max coins bursting all in (i,j) exclusive
public int MaxCoins(int[] nums)
{
    int n=nums.Length;
    int[] a=new int[n+2]; a[0]=a[n+1]=1;
    for (int i=0;i<n;i++) a[i+1]=nums[i];
    int N=n+2;
    int[,] dp=new int[N,N];

    for (int len=2;len<N;len++)
    for (int i=0;i+len<N;i++)
    {
        int j=i+len;
        for (int k=i+1;k<j;k++) // k = LAST burst in (i,j)
            dp[i,j]=Math.Max(dp[i,j], dp[i,k]+a[i]*a[k]*a[j]+dp[k,j]);
    }
    return dp[0,N-1];
}

// Next Greater Element — decreasing stack
public int[] NextGreater(int[] nums)
{

    int[] res=new int[n]; Array.Fill(res,-1);
    var st=new Stack<int>(); // stores indices

    for (int i=0;i<n;i++)
    {
        // pop all indices whose next greater is nums[i]
        while (st.Count>0 && nums[st.Peek()]<nums[i])
            res[st.Pop()]=nums[i];
        st.Push(i);
    }
    return res;
}

C#
public int LargestRectangleArea(int[] heights)
{
    var st=new Stack<int>();
    int maxA=0;
    int[] h=heights.Append(0).ToArray(); // sentinel

    for (int i=0;i<h.Length;i++)
    {
        while (st.Count>0 && h[st.Peek()]>h[i])
        {
            int height=h[st.Pop()];
            int width=st.Count==0 ? i : i-st.Peek()-1;
            maxA=Math.Max(maxA, height*width);

        st.Push(i);
    }

// Two-pointer O(1) space approach
public int Trap(int[] h)
{
    int left=0, right=h.Length-1, lMax=0, rMax=0, water=0;
    while (left<right)
    {
        if (h[left]<h[right])
        {
            lMax=Math.Max(lMax,h[left]);
            water+=lMax-h[left++];
        }
        else
        {
            rMax=Math.Max(rMax,h[right]);
            water+=rMax-h[right--];
        }
    }
    return water;
}

// Kth Largest — min-heap of size k
public int FindKthLargest(int[] nums, int k)
{
    var heap = new PriorityQueue<int,int>();
    foreach (int n in nums)
    {
        heap.Enqueue(n, n);
        if (heap.Count>k) heap.Dequeue(); // remove smallest
    }
    return heap.Peek(); // kth largest
}

// Top K Frequent — frequency as priority
public int[] TopKFrequent(int[] nums, int k)
{

    foreach (var (num,cnt) in freq)

Max-heap for lower half, min-heap for upper half. Keep sizes balanced (differ by at most 1). Median is always at the top of the larger heap, or average of both tops when equal size. Insertion: always push to max-heap first, then rebalance.
C#
public class MedianFinder
{
    // lo = max-heap (lower half), hi = min-heap (upper half)
    private PriorityQueue<int,int> _lo=new(), _hi=new();

    public void AddNum(int num)
    {
        _lo.Enqueue(num,-num); // negate for max-heap
        // balance: move lo's max to hi
        _hi.Enqueue(_lo.Peek(), _lo.Peek()); _lo.Dequeue();
        // if hi bigger, move hi's min back to lo
        if (_hi.Count > _lo.Count)
        { int v=_hi.Dequeue(); _lo.Enqueue(v,-v); }
    }

    public double FindMedian() =>
        _lo.Count>_hi.Count ? _lo.Peek() : (_lo.Peek()+_hi.Peek())/2.0;
}

public ListNode MergeKLists(ListNode[] lists)
{
    var heap = new PriorityQueue<ListNode,int>();
    foreach (var head in lists)
        if (head!=null) heap.Enqueue(head, head.val);

    var dummy=new ListNode(); var cur=dummy;

    while (heap.Count>0)
    {
        var node=heap.Dequeue();
        cur=cur.next=node;
        if (node.next!=null) heap.Enqueue(node.next, node.next.val);
    }

}

Path compression + union by rank. Path compression flattens the tree during Find. Union by rank attaches smaller tree under larger. Together: nearly O(1) per operation (inverse Ackermann).
C#
public class UnionFind
{
    private int[] _parent, _rank;
    public int Components { get; private set; }

    public UnionFind(int n)
    {
        _parent = Enumerable.Range(0,n).ToArray();
        _rank   = new int[n];
        Components = n;
    }

    // Path compression: flatten tree on the way up
    public int Find(int x)
    {
        if (_parent[x]!=x) _parent[x]=Find(_parent[x]);
        return _parent[x];
    }

    // Union by rank: smaller tree under larger
    public bool Union(int x, int y)
    {
        int px=Find(x), py=Find(y);
        if (px==py) return false; // already connected = cycle!
        if      (_rank[px]<_rank[py]) _parent[px]=py;
        else if (_rank[px]>_rank[py]) _parent[py]=px;
        else { _parent[py]=px; _rank[px]++; }
        Components--;
        return true;
    }

    public bool Connected(int x, int y) => Find(x)==Find(y);
}

// Detect undirected cycle: union returns false if same component
public bool HasCycle(int n, int[][] edges)
{
    var uf=new UnionFind(n);
    foreach (var e in edges)
        if (!uf.Union(e[0],e[1])) return true;
    return false;
}
Decision Guide

Use this pattern when

  • Dynamic connectivity — edges arrive one at a time
  • Undirected graph cycle detection
  • Count components remaining after grouping
  • Kruskal's MST — union edges greedily by weight

Prefer another when

  • Edge removal → use link-cut tree (very advanced)
  • Directed graph → DFS with 3 colors or Kahn's Topo Sort
  • Static graph (all edges known) → BFS/DFS simpler
  • Shortest path or weighted queries → Dijkstra/BFS
If Pattern Does Not Apply — Pivot Thinking

Decision Framework

Q1Need to also remove edges? → DSU only supports union. There's no efficient 'split'. Use a different approach or offline queries (process in reverse: deletions become additions).
Q2Directed graph connectivity? → DSU ignores edge direction. For directed reachability, use DFS. For strongly connected components, use Tarjan's or Kosaraju's algorithm.

Compute in-degrees. Enqueue all nodes with in-degree 0. Loop: dequeue, add to result, decrement neighbors' in-degrees, enqueue those that reach 0. If result.Count < n, a cycle exists — return empty.
C#
public int[] FindOrder(int n, int[][] prereqs)
{
    var adj = new List<int>[n];
    int[] inDeg = new int[n];
    for (int i=0;i<n;i++) adj[i]=new List<int>();

    foreach (var p in prereqs)
    { adj[p[1]].Add(p[0]); inDeg[p[0]]++; }

    var q=new Queue<int>();
    for (int i=0;i<n;i++) if (inDeg[i]==0) q.Enqueue(i);

    var res=new List<int>();
    while (q.Count>0)
    {
        int node=q.Dequeue(); res.Add(node);
        foreach (int next in adj[node])
            if (--inDeg[next]==0) q.Enqueue(next);
    }

    return res.Count==n ? res.ToArray() : Array.Empty<int>();
    // Empty = cycle detected
}
Run DFS, pushing each fully-processed node onto a stack (postorder). Reverse the stack for topological order. Color: 0=unvisited, 1=in-progress (GRAY), 2=done. If you hit GRAY, there's a cycle.
C#
public int[] TopoSortDfs(int n, int[][] edges)
{
    var adj=new List<int>[n];
    for (int i=0;i<n;i++) adj[i]=new List<int>();
    foreach (var e in edges) adj[e[0]].Add(e[1]);

    int[] state=new int[n]; bool cycle=false;
    var stack=new Stack<int>();

    for (int i=0;i<n;i++) if (state[i]==0) Dfs(i);
    return cycle ? Array.Empty<int>() : stack.ToArray();

    void Dfs(int u)
    {
        if (cycle) return;
        state[u]=1;
        foreach (int v in adj[u])
        {
            if (state[v]==1) { cycle=true; return; }
            if (state[v]==0) Dfs(v);
        }
        state[u]=2; stack.Push(u); // push after all descendants
    }
}

Each TrieNode stores children (Dictionary or array[26]) and an IsEnd flag. Insert: create nodes level by level. Search: traverse nodes. StartsWith: traverse and check if path exists.
C#
public class TrieNode
{
    public TrieNode[] Children = new TrieNode[26];
    public bool IsEnd;
}

public class Trie
{
    private readonly TrieNode _root = new();

    public void Insert(string word)
    {
        var node = _root;
        foreach (char c in word)
        {
            int i = c - 'a';
            node.Children[i] ??= new TrieNode();
            node = node.Children[i];
        }
        node.IsEnd = true;
    }

    public bool Search(string word)
    {
        var node = GetNode(word);
        return node!=null && node.IsEnd;
    }

    public bool StartsWith(string prefix) => GetNode(prefix)!=null;

    private TrieNode? GetNode(string s)
    {
        var node=_root;
        foreach (char c in s)
        {
            node=node.Children[c-'a'];
            if (node==null) return null;
        }
        return node;
    }
}
Build a trie from all words. DFS the grid. At each cell, follow the trie node — if no child exists, prune immediately. When IsEnd is true, record the word and set IsEnd=false to avoid duplicates. Remove word from trie once found to continue pruning.
C#
public IList<string> FindWords(char[][] board, string[] words)
{
    var root=new TrieNode();
    foreach (var w in words)
    {
        var n=root;
        foreach (char c in w)
        { n.Children[c-'a']??=new TrieNode(); n=n.Children[c-'a']; }
        n.IsEnd=true;
    }

    var res=new List<string>();
    int m=board.Length, n2=board[0].Length;

    void Dfs(int r, int c, TrieNode node, StringBuilder sb)
    {
        if (r<0||r>=m||c<0||c>=n2) return;
        char ch=board[r][c];
        if (ch=='#'||node.Children[ch-'a']==null) return;
        node=node.Children[ch-'a'];
        sb.Append(ch); board[r][c]='#';
        if (node.IsEnd) { res.Add(sb.ToString()); node.IsEnd=false; }
        Dfs(r+1,c,node,sb); Dfs(r-1,c,node,sb);
        Dfs(r,c+1,node,sb); Dfs(r,c-1,node,sb);
        board[r][c]=ch; sb.Remove(sb.Length-1,1);
    }

    for (int r=0;r<m;r++) for (int c=0;c<n2;c++)
        Dfs(r,c,root,new StringBuilder());
    return res;
}
